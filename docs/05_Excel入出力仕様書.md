# スキル秘伝HP Excel入出力仕様書

## 1. 目的

利用者の所持秘伝・編成をバックアップし、別PCまたは後日に復元できるようにする。

元マスタExcelと、利用者が書き出すバックアップExcelは別形式にする。

## 2. ファイルの種類

### マスタExcel

- ファイル例: `スキル秘伝_v0.1_PN.xlsx`
- 管理者がクラス、スキル、固定枠、効果を更新するための元資料
- 公開サイトの起動時に直接読むのではなく、公開前に検証してアプリ用マスタへ変換する

### ユーザーバックアップExcel

- ファイル例: `skill-sigil-backup_20260712_153000.xlsx`
- 所持品、編成、装着状況、バージョンを保存
- 利用者が手元へ保管する

## 3. バックアップExcelのシート

### README

| 列 | 内容 |
| :---- | :---- |
| format_version | Excel形式の版 |
| schema_version | 保存データ構造の版 |
| master_version | マスタの版 |
| exported_at | 書き出し日時 |
| app_name | サイト名 |
| notice | 手編集時の注意 |

### 所持秘伝

| 列名 | 必須 | 内容 |
| :---- | :---- | :---- |
| inventory_id | 必須 | 所持品ID |
| 秘伝タイプID | 必須 | タイプID |
| 秘伝タイプ名 | 必須 | 日本語表示名 |
| 効果ID | 必須 | 効果ID |
| 効果名 | 必須 | 日本語表示名 |
| 等級 | 必須 | 深淵・太古・混沌 |
| 数値 | 任意 | 自由入力値 |
| 所持数 | 必須 | 1以上の整数 |
| メモ | 任意 | ユーザーメモ |
| 更新日時 | 必須 | ISO形式日時 |

### 編成

| 列名 | 必須 | 内容 |
| :---- | :---- | :---- |
| build_id | 必須 | 編成ID |
| 編成名 | 必須 | ユーザー入力名 |
| class_id | 必須 | クラスID |
| クラスコード | 必須 | WRなど |
| 作成日時 | 必須 | ISO形式日時 |
| 更新日時 | 必須 | ISO形式日時 |

### 装着状況

| 列名 | 必須 | 内容 |
| :---- | :---- | :---- |
| build_id | 必須 | 編成ID |
| skill_id | 必須 | スキルID |
| グループ | 必須 | special / normal |
| 表示番号 | 必須 | グループ内番号 |
| slot_no | 必須 | 1〜4 |
| inventory_id | 必須 | 所持品ID |

### マスタ参照

- 使用したクラス・スキル・固定枠の最小情報
- 古いバックアップを調査するために保持
- 直接編集対象にはしない

## 4. 書き出し

1. 現在データの整合性を確認する。
2. 形式バージョンを付ける。
3. 全編成と所持品を書き出す。
4. ファイル名へ日時を付ける。
5. 完了後に `last_export_at` を更新する。

装着数が所持数を超えている場合は、エラーを表示して書き出しを停止する。

## 5. 読み込み

### 事前確認

- `.xlsx` 形式か
- READMEシートがあるか
- `format_version` を読めるか
- 必須シート・必須列があるか
- IDが重複していないか
- 所持数が1以上か
- 等級が許可値か
- 固定枠と秘伝タイプが一致するか
- 装着数が所持数を超えていないか

### 読み込みモード

#### 置換

- 現在データを退避した後、読込データへ置き換える。
- もっとも確実な復元方法として推奨する。

#### 追加統合

- IDが異なる所持品・編成を追加する。
- 同じIDで内容が異なる場合は、読込前に選択を求める。
- 同じタイプ・効果・等級・数値を自動で数量加算しない。候補として提示する。

## 6. エラー区分

| 区分 | 例 | 動作 |
| :---- | :---- | :---- |
| エラー | 必須列なし、壊れた数量、参照先なし | 読み込み不可 |
| 警告 | 古いマスタ、無効化された効果、未知のクラス | 確認後に保持可能 |
| 情報 | 名称だけ変更、追加メモあり | 読み込み可能 |

## 7. バージョン互換

- `format_version` が新しすぎる場合は、更新版サイトを案内する。
- 古い形式は移行処理がある場合だけ読み込む。
- 元のファイルは変更せず、変換後の新ファイルを別名保存する。

## 8. 手編集

- 所持数、数値、メモ、編成名は手編集可能。
- ID、クラスコード、スキルID、枠番号は原則変更しない。
- 入力規則とREADMEで編集可能列を明示する。

## 9. プライバシー

バックアップにはゲームID、パスワード、メールアドレスを含めない。利用者が入力した所持品・編成・メモだけを保存する。

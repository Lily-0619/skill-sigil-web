# 17 編成比較機能 設計方針

作成: 2026-07-23 ／ 依頼者: まな ／ 状態: 方針確定・実装未着手

## 目的

編成管理にある複数の編成を並べて、**どのスキルに・どの秘伝の・どの効果がついているか**を
突き合わせて比較できる画面を追加する。クラスは統一しない（別クラス同士でも並べられる）。

秘伝の数値は**合算しない**。同じ効果が重複しても足し算せず、既存の「つけている秘伝一覧」と
同じく個数（×N個）でまとめる。比較はあくまで「何がどれだけ付いているか」を見るためのもの。

## 何を作るか（スコープ）

左ナビに「比較 / Compare」を追加し、編成管理にある編成同士を比較する読み取り専用画面を作る。
表示は3つのビューをトグルで切り替える。

- **案A スキルごと** — スキル → 固定4枠 → 各枠の秘伝（効果名・等級・数値）を縦に並べる。
  「3番スキルの2枠目が混沌の命中+22」まで枠単位で分かる。密度が高く、細かく突き合わせる用。
- **案B タイプ別サマリー（集計）** — 秘伝タイプ → 効果 → 等級で、付いている本数を集計する。
  数値は既存どおり「値 ×N個」で個数集約。どんな傾向で積んでいるかを把握する用。枠位置は出さない。
- **案C グラフ** — 案Bの集計データを元に、編成ごとの本数をグラフで見せる（例: 秘伝タイプ別の
  本数を横棒で比較 / 等級別の本数を積み上げ）。傾向差を視覚的に掴む用。

## スコープ外（今回作らない）

- スキル秘伝をまとめて画像化する機能（別途検討。DOM→画像化のため技術検証が必要）。
- 比較画面からの編集。比較は読み取り専用とし、編集は編成編集・編成管理で行う。
- 数値の合算・スコア化・優劣判定。合算しない方針のため実装しない。

## 画面と入口

### ナビ

`src/components/Shell.tsx` の `NAV` に項目を追加する。位置は「編成管理（Manage）」の下。

```
{ id: "compare", ja: "比較", en: "Compare" }
```

- `Screen` 型に `"compare"` を追加（`src/components/Shell.tsx`）。
- クラス未選択でも開ける（`needsClass` は付けない。編成管理と同じ扱い）。
- `src/App.tsx` の画面分岐に `screen === "compare"` を追加し `<Compare />` を表示する。

### 入口フロー

「比較」画面を開くと、上部に**編成の複数選択エリア**（編成管理にある全編成をチェックで選ぶ）を出し、
2つ以上選ぶと下に比較結果が並ぶ。編成管理画面そのものは改修しない（Myモード/Freeモードや
削除・複製と混ざらないよう、選択は比較画面側に閉じる）。

- 選択0〜1個のときは結果を出さず、「2編成以上を選んでください」の空状態を表示。
- 選択は全クラス横断・My/Free混在で可。各カラムに MY/FREE バッジとクラス名を表示。
- 横並びの**最大は4編成**（既定）。5個以上は可読性が落ちるため選択不可にする。最小2編成。
  ※上限は運用して見直す余地あり。まずは4で固定。

## データの作り方

既存ロジックを再利用し、同じ計算を複製しない（`docs/16-2` の原則に従う）。

- 集計の中核は `src/logic/equip.ts` に共通関数として置く。`BuildEdit.tsx` の `equippedCatalog`
  相当（タイプ→効果→等級→値の集約、`collapseValues` の ×N個ロジック）を関数として切り出し、
  比較画面と編成編集の両方から呼ぶ。
- 案A/B/グラフはすべて**同じ集計結果**を入力に描く（ビューが違うだけでデータ源は一つ）。
- Myは `equips` → `inventory` を辿って効果・等級・数値を得る。Freeは `freeEquips` を直接参照。
  この分岐は既存 `equippedCatalog` と同じにする。
- 枠数カウント（「装着 x/68枠」）は既存 `buildEquipStats`（`used` / `total`）を流用する。
  分母は秘伝対象スキル（`sigil_eligible`）× 4枠。これは枠の数の集計であり、数値の合算ではない。
- 数値なし秘伝（`valueless`）は「数値なし」表示（既存踏襲）。

## グラフ（案C）の方針

- **外部ライブラリは追加しない。** 依存は現状 `react` / `react-dom` / `xlsx` のみで、この方針を維持する。
  グラフは軽量な自前実装（div幅バー、または素のSVG）で描く。凝ったチャートは目的外。
- 初期実装は「編成ごとの、秘伝タイプ別の本数」を横棒で並べる程度から始める。
  余力があれば「等級別（混沌/太古/深淵）の本数」の積み上げを追加。
- 色は既存のタイプ色・等級色（`master.json` の `sigil_types[].color` / `rarities[].color`）を使う。

## コンポーネント構成（予定）

`docs/16-2` の分割方針に合わせ、画面組み立てと表示部品を分ける。

```
src/components/
  Compare.tsx                     # 画面全体の組み立て・選択状態・ビュー切替
  compare/
    BuildSelectBar.tsx            # 上部の編成マルチセレクト（2〜4件）
    CompareBySkill.tsx            # 案A スキルごと
    CompareBySummary.tsx          # 案B タイプ別サマリー（集計）
    CompareChart.tsx              # 案C グラフ（自前バー/SVG）
src/logic/equip.ts                # 集計関数を追加（BuildEdit と共用）
```

## 見た目

- 既存アプリの配色に合わせる（濃紺パネル `--panel` ＋ピンク `--pink`、等級・タイプチップ）。
- カラムは横並び、幅が足りなければ横スクロール。1カラム＝1編成。
- ヘッダーに「◀ 編成管理へ戻る」相当（ナビで戻れるため任意）、ツールバーにビュー切替トグル。
- モックイメージは会話内で提示済み（案A＝スキルごと4枠 / 案B＝タイプ別サマリー）。数値はダミーで、
  実装では実データ（`master.json` の `name_ja`、装着済みの `rarity`・`value_text`）に差し替わる。

## テスト

- 集計関数（切り出したもの）に対し、`npm test`（vitest）へユニットテストを追加する。
  - My編成・Free編成それぞれで、タイプ→効果→等級→本数が正しく出ること。
  - 同一効果＋等級の重複が「値 ×N個」に集約され、**合算されない**こと。
  - 別クラス2編成を並べても各カラムが独立して集計されること。
- 既存テスト（装着ルール・Excel往復・スモーク）を壊さないこと。

## 未確定・見直し余地

- 横並び上限（既定4）。運用して増減を検討。
- グラフの軸（タイプ別のみ / 等級別も出すか）。まずタイプ別本数から。
- 案A/B/グラフを3つ全部トグルにするか、案A＋案B＋グラフの縦積みにするか。初期はトグルで実装。

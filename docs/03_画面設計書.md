## 2. 共通レイアウト

- 上部: サイト名、現在のクラス、編成名、自動保存状態
- 左側: クラス・画面切替
- 中央: 現在の主要操作
- 右側: 所持秘伝トレイまたは詳細
- 下部: バージョン、非公式表記、公式リンク

背景にはキャラクター画像を薄く表示し、コンテンツ領域には半透明または不透明の面を敷く。

## 3. S-01 クラス選択

### 表示

- クラスアイコン
- クラス略称（例: WR）をクラスボタン下に表示
- クラス名（ゲーム内クラス名をカタカナ音写。例: WR＝ウォーリア）
- 保存編成数
- 最終更新日時
- 検索

### 表示クラスの範囲（2026-07-13決定）

- `class_button` にボタンはあるがスキルデータを持たないクラス（例: UC）は、現時点では非表示にする。
- クラスが正式追加され、スキルデータが揃った時点で表示対象へ切り替える（`classes.enabled` で制御。詳細は`04_データ設計書.md`）。
- 略称表示・カタカナ名表示は、将来クラスが増えた場合も同じ規則で追加できるようにする。

### 背景

- `画像/character/` 全体から起動時に1枚をランダム選択
- 直前と同じ画像は、候補が2枚以上なら避ける
- 右下または左右どちらかへ寄せ、人物が切れすぎない配置
- 目安透明度: 8〜14%

### 操作

- クラスボタンをクリックするとS-02へ移動
- 初回は所持秘伝登録への案内を表示

## 4. S-02 編成編集

### 構成

```text
┌ 上部: クラス / 編成 / 保存状態 ─────────────────────┐
│ 特別スキル（ラバムスキル） 1  2  3  4                 │
│ 通常スキル 1〜13                                     │
│   [選択スキル詳細] [固定4枠]                         │
│                                  [所持秘伝トレイ]     │
└───────────────────────────────────────────────┘
```

### スキルカード

- アイコン
- 表示番号
- スキル名
- 装着済み数 `0/4`〜`4/4`
- 未装着警告
- 下段13番のみ「秘伝対象外」

### 固定4枠

- 枠番号1〜4
- 固定秘伝タイプ
- 装着効果
- 等級
- 数値
- 解除ボタン

### 所持秘伝トレイ

- タイプ、等級、効果検索
- 所持数、使用中、残り
- 装着可能な候補を上へ表示
- 残り0は選択不可だが、現在の装着先を確認できる

### 装着操作

1. 所持秘伝をクリックする。
2. 対応枠だけを強調する。
3. 枠をクリックして装着する。
4. 残数を即時更新する。

別スキルで使用中の最後の1個を選んだ場合は、現在の装着先を表示し、移動確認を行う。

## 5. S-03 所持秘伝管理

### 一覧列

- 秘伝タイプ
- 効果
- 等級
- 数値
- 所持数
- 使用中
- 残り
- メモ

### 登録フォーム

利用者が迷わず登録できることを最優先にする。フォームは上から順に入力し、各項目は前の入力に応じて選択肢を絞り込む。

| 順 | 項目 | 入力方式 | 選択肢・初期値 | 必須 |
| :---- | :---- | :---- | :---- | :---- |
| 1 | 秘伝タイプ | アイコン付きボタン群（7種） | 系列・守護・無欠・鮮明・整った・微か・煌めく（`sigil_types`） | 必須 |
| 2 | 効果 | 選択＋検索（1で絞込済み） | 選択したタイプに属する `effect_master` の `enabled=true` のみ表示。煌めくは混沌のみ選べるようグレーアウトで案内 | 必須 |
| 3 | 等級 | 3択ボタン（深淵/太古/混沌） | 色チップ付き。2で選んだ効果にその等級の値が無い（`-`）場合はボタンを無効化し理由を表示 | 必須 |
| 4 | 数値 | テキスト入力（自動初期値＋手編集可） | 1〜3の組み合わせから `effect_master` の既定値（`value_*_a` / `value_*_b`、無欠は2値同時表示）を自動入力。空欄化・上書き可 | 任意 |
| 5 | 所持数 | 数値ステッパー | 初期値1、1以上の整数 | 必須 |
| 6 | メモ | テキスト入力 | 空欄 | 任意 |

補足:

- 数値が「A / B」形式の効果は、初期表示をラジオボタン（Aを選ぶ／Bを選ぶ）にし、選んだ方の値を4へ自動反映する。両方を残したい場合はメモへ記録するよう案内文を出す。
- 無欠の2値同時効果（例: `20 , 3%`）は、数値欄を2つのサブ入力（効果1・効果2）に分けて表示する。
- 系列タイプは深淵のみ初期値を自動入力し、太古・混沌は空欄のまま手入力を促す注記を表示する（`04_データ設計書.md` 5節参照）。
- 登録ボタンは1〜3、5が揃うまで無効化する。4・6は未入力でも登録できる。

同じタイプ・効果・等級・数値（4の内容が完全一致）なら統合候補を提示し、「所持数を合算する」か「別行として保持する」かを利用者に選ばせる。数値が異なる場合は自動的に別行で管理する。

## 10. キャラクター背景

- クラス選択前: 全画像からランダム
- クラス選択後: 選択クラスに対応する画像だけからランダム
- 画面を開くたびに選び直す
- 画面内の再描画では不用意に切り替えない
- 読込失敗時はグラデーションへ切り替える
- 背景はクリック不可・読み上げ対象外

## 11. アニメーション仕様

### 目的・原則

- 操作の理解を助け、サイトの世界観（黒い砂漠モバイルの雰囲気）を演出するために使う。操作を妨げる長さ・派手さにしない。
- 同じ種類の要素には同じアニメーションを使い、画面ごとに演出をばらつかせない。
- OS・ブラウザの「アニメーション削減（`prefers-reduced-motion: reduce`）」設定を検知した場合は、すべてのアニメーションを即時表示（duration実質0）へ切り替える。
- 初回表示時のみ動かす（一覧の再描画・タブ切替の毎回では動かさない）。データ更新による再描画は対象外とし、新規追加・新規表示の要素のみアニメーションさせる。
- 装着・解除・削除など「操作結果の確定」を示す動きは、演出用アニメーションと区別し、常に有効にする（削減設定時も短い即時フィードバックは残す）。

### 共通パラメータ

| 項目 | 既定値 | 備考 |
| :---- | :---- | :---- |
| 標準duration | 400ms | 見出し・カード単体 |
| 短いduration | 200ms | ボタン・アイコン・トースト通知 |
| 長いduration | 600ms | 背景・ページ全体の切り替え |
| easing（出現） | `ease-out`（`cubic-bezier(0.16, 1, 0.3, 1)`） | 減速して止まる、要素が「収まる」印象 |
| easing（退出） | `ease-in` | 加速して消える |
| stagger（連続要素の遅延） | 40〜60ms/件、最大8件分まで | カード一覧・スキルカードなど。9件目以降は遅延を増やさず同時発火 |
| 初期transform量（スライド） | 16〜24px | 大きすぎるとガタつきの印象になるため小さめに統一 |
| ブラー初期値 | 6〜10px | 大きすぎる値は文字がにじんで見えるため上限を設ける |

### パターン定義

| パターン名 | 変化内容 | 主な用途 |
| :---- | :---- | :---- |
| フェードイン | `opacity: 0 → 1` | 背景キャラクター画像、詳細パネルの中身 |
| スライドイン（下から） | `opacity: 0 → 1` ＋ `translateY(20px → 0)` | カード一覧、フォーム項目、通知トースト |
| スライドイン（横から） | `opacity: 0 → 1` ＋ `translateX(±20px → 0)` | サイドパネル（所持秘伝トレイ）、モーダルの左右移動 |
| ブラーイン | `opacity: 0 → 1` ＋ `filter: blur(8px → 0)` | 見出し・キャッチコピーなど文字要素、クラス選択画面の背景キャラクター |
| フェード＋スライド＋ブラー（複合） | 上記3種の組み合わせ | サイトタイトル、S-01のクラス名など「見せ場」にしたい文字 |

複合パターンは多用せず、画面につき1〜2箇所（タイトル・主要見出し）に限定する。同一画面内でパターン数を増やしすぎるとうるさく見えるため、1画面あたり基本2種類（例: 見出し＝ブラーイン、カード群＝スライドイン）までに絞る。

### 要素別の適用一覧

| 要素 | 画面 | パターン | 発火条件 |
| :---- | :---- | :---- | :---- |
| サイトタイトル・見出し | 全画面共通 | ブラーイン（+わずかなスライド） | ページ読み込み時、1回のみ |
| 背景キャラクター画像 | S-01 / S-02 | フェードイン（長いduration） | 画像決定後にフェード表示。切り替え時も同様 |
| クラスボタン一覧 | S-01 | スライドイン（下から）＋stagger | 一覧表示時 |
| スキルカード一覧 | S-02 | スライドイン（下から）＋stagger | 編成編集画面表示時 |
| 固定4枠パネル | S-02 | フェードイン | スキルカード選択時に展開 |
| 所持秘伝トレイ | S-02 / S-03 | スライドイン（横から） | 画面表示時、または開閉トグル時 |
| 装着成功・解除・削除の確定表示 | S-02 / S-03 / S-04 | 短いフェード（duration 200ms、削減設定でも常時有効） | 操作確定の都度 |
| 警告・エラーメッセージ | 全画面共通 | スライドイン（下から、短いduration） | 発生の都度 |
| モーダル（削除確認・移動確認など） | 全画面共通 | フェード＋わずかなスケール（0.96→1） | 表示・非表示の都度 |
| バックアップ結果一覧 | S-05 | スライドイン（下から）＋stagger | 検証・取込完了時 |

### 実装ガイド（設計メモ）

- 一覧・カードのように画面内スクロールで現れる要素は `IntersectionObserver` などで表示検知し、初めて画面内に入った時だけ再生する（過去に再生済みの要素は再度発火させない）。
- CSSの `transition` / `animation` を基本とし、JS制御は「表示検知」「削減設定の分岐」「stagger遅延の計算」に限定する。
- パフォーマンスのため、アニメーション対象プロパティは `opacity` / `transform` / `filter` に限定し、レイアウトを再計算させる `width` / `height` / `top` / `left` は使わない。
- 背景画像のフェードは、画像読込前に暗い単色または既定のグラデーション（10章）を敷いておき、読込完了後にクロスフェードする。読込失敗時はフェードせず即座にグラデーションへ切り替える。

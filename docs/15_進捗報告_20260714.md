## 2. 運用上の制約・注意点(引き継ぎ用)

- Cowork環境の`mcp__workspace__bash`サンドボックスには、まなが選択したDドライブの
  フォルダが基本的にマウントされない。Read/Write/Edit/Globなどのファイルツールは
  直接読み書きできるが、`npm install`/`npm test`/`npm run build`のような**実行を伴う
  操作はサンドボックス側から完結できない**。
- 代わりに`mcp__computer-use__*`でまなの実機を直接操作し、コマンド プロンプトへ
  コマンド文字列をクリップボード経由で渡し、まなに「右クリック→貼り付け→Enter」を
  してもらう運用を確立した。
- Claudeが作成した`.bat`ファイルは、まなのPCのセキュリティ設定によって直接実行が
  ブロックされる(組織のセキュリティポリシー/EDRとみられる)。このため上記の
  クリップボード方式を標準の実行手段とする。
- 一度、`スキル秘伝HP.html`が原因不明のロックで上書き・削除ともにできなくなる事象が
  発生。まなのPC再起動で解消した。再発した場合はまず再起動を試すこと。

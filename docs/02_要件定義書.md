# スキル秘伝HP 要件定義書

## 1. 前提

- 日本語のみをMVP対象とする。
- PCブラウザで利用する。
- 最新版のChrome・Edgeを主対象とする。
- サイトの閲覧は無料、ログイン不要。
- ユーザーデータはPC内に保存し、サーバーへ送信しない。
- 現在のマスタは1クラス17スキル。ただし件数をプログラムへ固定しない。

## 2. ゲームルール

### REQ-RULE-001 スキル構成

- 上段1〜4: 特別なスキル
- 下段1〜13: 通常スキル
- 合計: 17スキル
- 下段13番は秘伝装着不可
- 秘伝装着可能: 16スキル

### REQ-RULE-002 秘伝枠

- 装着可能な各スキルは固定4枠を持つ。
- 枠ごとの秘伝タイプはマスタで固定する。
- ユーザーは枠のタイプを変更できない。
- 各枠へ選べる効果は、枠と同じ秘伝タイプの効果だけ。

### REQ-RULE-003 所持数

- 同じ所持秘伝1個を、同じ編成内の複数スキルへ重複装着できない。
- 同じ秘伝を2個所有していれば、2つのスキルへ1個ずつ装着できる。
- 残数は `所持数 - 同じ編成内の使用数` で求める。
- 所持数を減らした結果、使用数を下回る場合は保存前に解消を求める。

### REQ-RULE-004 等級

- 深淵: 赤
- 太古: 濃いピンク
- 混沌: 深い青
- 等級は必須選択。

### REQ-RULE-005 数値

- 効果数値のマスタが判明済み（`11_効果マスタ数値.md`）。効果と等級を選ぶと既定値が自動で入る。
- 既定値はユーザーが後から編集・空欄化できる（自由入力の文字列として保持）。
- `%`、秒、回、固定値などを入力できる。
- 効果によっては「A / B」の2値を持つ（同一等級内で取りうる下位/上位のいずれか）。無欠のカンマ区切りのみ2効果の同時付与。等級に該当値が無い場合は選択不可（煌めくは混沌のみ）。
- 系列タイプは深淵10%のみ判明、太古・混沌は要確認。

## 3. 機能要件

| ID | 機能 | 要件 |
| :---- | :---- | :---- |
| F-001 | クラス選択 | クラスアイコン・クラス名を一覧表示し、対象クラスへ移動する |
| F-002 | スキル表示 | 上段4、下段13をグループ別に表示する |
| F-003 | 対象外表示 | 下段13番を「秘伝対象外」として表示する |
| F-004 | 所持品登録 | タイプ・効果・等級・数値・所持数・メモを登録する |
| F-005 | 所持品検索 | タイプ・効果・等級・使用状況で絞り込む |
| F-006 | 秘伝装着 | 所持秘伝を選び、互換性のある固定枠へ装着する |
| F-007 | 装着解除 | 装着済み秘伝を外し、残数へ戻す |
| F-008 | 付け替え | 別スキルへ移動する際、現在の装着先を示して確認する |
| F-009 | 残数表示 | 所持数・使用中・残りを常時表示する |
| F-010 | 編成保存 | クラスごとに複数の編成を保存する |
| F-011 | 自動保存 | 編集ごとにPC内へ保存する |
| F-012 | Excel出力 | 所持品・編成・装着・バージョンをExcelへ出力する |
| F-013 | Excel読込 | Excelから置換または追加統合で復元する |
| F-014 | 読込検証 | 不正な列、ID、タイプ、数量、バージョンを取込前に表示する |
| F-015 | 背景演出 | 起動ごとにキャラクター背景をランダム表示する |
| F-016 | ヘルプ | 保存場所・Excel復元・非公式サイト表記を説明する |

## 4. 非機能要件

### 操作性

- マウス操作を主とし、Pointer Eventsでタッチ操作にも備える。
- ドラッグ操作だけに依存せず、クリック選択→枠クリックでも装着できる。
- 選択中、装着可能、装着不可、使用済み、未装着を色と文字の両方で示す。

### 可読性

- 1440px幅を基準に設計し、1024px以上で主要操作が破綻しない。
- 背景画像は装飾扱いとし、文字やボタンのコントラストを優先する。
- 等級は色だけでなく名称も必ず表示する。

### 性能

- 初回表示に必要な画像だけを読み込む。
- スキル画像・背景画像は遅延読み込みする。
- 画像読込失敗で操作不能にしない。

### データ保護

- ゲームID、パスワード、メールアドレスは扱わない。
- ユーザーデータを外部送信しない。
- ブラウザデータ削除で消える可能性を明示し、Excelバックアップを案内する。

### 保守性

- クラス、スキル、固定枠、効果、画像はデータファイルで管理する。
- `schema_version` と `master_version` を分ける。
- 画像追加時に手作業でソースコードへ一覧追記しない。

## 5. 保存要件

- PC内の保存先はIndexedDBを第一候補とする。
- 自動保存は編集完了単位で行う。
- 保存失敗時は画面上へ警告する。
- Excel出力を手動バックアップとして扱う。

## 6. 公開要件

- フッターに非公式ファンサイト表記を置く。
- 公式サイトへのリンクを置く。
- Pearl Abyssのロゴ・商標を改変しない。
- MVPは広告・課金なし。

## 7. マスタ更新要件

- 現在は17スキル表示を正とする。
- 公式アップデートで件数が変わった場合、マスタ変更で対応できる。
- マスタ取込時にクラス数、スキル数、ID重複、固定枠数を検証する。
- 古い編成に存在しないスキルがあっても即削除せず、警告つきで保持する。

## 3. TOP: 左「Myスキル秘伝」／右「Freeスキル秘伝」(新機能)

### 人間向け説明

いま作ってあるのは「自分が持っている秘伝だけで組む」モード。これを**My**と呼ぶ。
新しく作る**Free**は、所持登録ゼロでも、枠に合う秘伝を自由にはめて「理想形」「目指すセット」を
組めるモード。ゲームを始めたばかりでも、目標の編成を先に設計できる。

### 仕様

- トップの左パネル: タイトル「Myスキル秘伝」（EN: My Sigil Build）→ 現行アプリへ。
- 右パネル: タイトル「Freeスキル秘伝」（EN: Free Sigil Build）→ Freeモードで入る。COMING SOONは廃止。
- Freeモードの編成編集:
  - 所持秘伝トレイの代わりに**秘伝カタログ**を表示（マスタ全効果。タイプ・等級・検索で絞り込み）。
  - 枠タイプ一致のみ検証。**所持数・残数・重複の制限なし**。
  - 数値はマスタ固定値（2値効果は下位/上位を選択、#6と同じ方式）。
  - ヘッダー・編成カードに「FREE」バッジを表示し、Myと視覚的に区別（淡ピンク枠など）。
- 編成管理(#7)では My/Free 混在の一括表示＋バッジ区別。
- Excel出力は**Myと分ける**: 既存シート(所持秘伝/編成/装着状況)はMy専用のまま、`Free編成`・`Free装着状況`シートを追加。`format_version` を 2 へ。v1形式の読込互換は維持（Freeシートなし=Free編成0件として読む）。

### データ設計

```
builds に mode: "my" | "free" を追加（既存データは読込時に "my" を補完 = migration）
free_equips（新設）: build_id / skill_id / slot_no / effect_id / rarity / value_text
  ※ Myの equipped_sigils は inventory_id 参照、Freeは効果を直接参照する点が本質的な違い
```

- `src/types.ts`: `Build.mode`、`FreeEquip` 型、`UserData.freeEquips` を追加。
- `src/state/store.tsx`: `FREE_EQUIP` / `FREE_UNEQUIP` アクション、LOADED時のmigration。
- `src/logic/equip.ts`: `canEquipFree()`（タイプ一致＋効果enabledのみ）。
- `src/logic/excel.ts`: v2形式の書き出し・読込・検証（枠タイプ一致検証はFreeにも適用）。

### なぜこの設計か

Freeを「所持数∞のダミー在庫」で表現する案もあるが、在庫画面にダミーが混ざり
Excelも汚れる。**編成側で参照先を分ける**方が、My/Freeの出力分離・将来の
「Freeで組んだ編成に必要な不足秘伝リスト」機能（将来版候補）にも素直につながる。

### 完了条件

- 所持登録0件のままFree編成が組める／Myでは従来どおり残数制限が効く。
- Excel書き出し→読み込みでMy/Freeが正しく往復する（テスト追加）。
- トップ両パネルとも hover側のみ映像が動く挙動は維持。

## 4. バグ修正: クラス選択直後にスキルアイコンが表示されない

- 症状: クラス選択→編成編集の初回表示でスキルカードが出ない。別タブへ行って戻ると表示される。
- 原因（特定済み）: `src/hooks/useReveal.ts` が `useRef` + 空依存 `useEffect` のため、
  `BuildEdit` 初回レンダー時（初期編成の自動作成前は `return null`）に要素が無く、
  IntersectionObserver が一度もattachされない。その後要素がマウントされても再attachされず、
  `.rv`（opacity:0）のまま残る。
- 修正: useReveal を **callback ref 方式**へ変更する。

```ts
// useReveal 修正イメージ
const [el, setEl] = useState<T | null>(null);
useEffect(() => {
  if (!el || played.current) return;
  // …IntersectionObserver attach（現行ロジックのまま）
}, [el]);
return { ref: setEl, cls: inView ? "rv in" : "rv" };
```

- 呼び出し側（`ref={grid1.ref}` など）は変更不要。`Shell` / `ClassSelect` / `Inventory` / `Builds` / `Backup` / `Help` にも同フックが使われているため、全画面で初回表示を確認する。
- 完了条件: クラス選択→直後の編成編集でスキルアイコンが即表示される。スモークテストに「S-02初回表示でスキルカードが可視」の検証を追加。

## 5. 編成編集: スロット先行選択フロー

- 背景: ゲーム内はスロットを先に選ぶ操作なので、それに合わせる。
- 新フロー（主）: スキルカード選択 → **枠(スロット)をクリック** → `selectedSlot` 状態に →
  所持秘伝トレイのうち「枠タイプ一致かつ残数>0」のアイテムが**明るく強調**（それ以外は減光）→
  アイテムクリックで即装着 → 選択解除。
- 既存フロー（従）: アイテム先行→枠クリックも残す（両方向対応）。`selectedItem` と `selectedSlot` は排他（片方を選んだらもう片方はクリア）。
- 実装: `src/components/BuildEdit.tsx` に `selectedSlot: {skillId, slotNo} | null` を追加。
  トレイ側は `canEquip()` の判定結果で `lit` / `dim` クラスを付与。
  `src/styles/global.css` に `.slot-cell.selected`（ピンク強枠）、`.tray-item.lit`（明るく＋グロウ）、`.tray-item.dim`（opacity 0.35）を追加。
- 装着済み枠をクリックした場合もそのまま選択でき、別アイテムをクリックしたら置き換え確認（既存モーダル流用）。
- 完了条件: スロットクリック→トレイ強調→アイテムクリックで装着が完了する。移動確認・置き換え確認は従来どおり動く。Freeモード(#3)のカタログにも同じ強調を適用。

## 6. 所持秘伝: 数値を編集不可に

- 背景: 効果数値はマスタで固定と確定したため、自由入力は不要（誤入力防止）。
- 変更（`src/components/Inventory.tsx`）:
  - 手入力 `<input>` を廃止し、自動決定された値の**読み取り専用表示**へ（チップ表示）。
  - 2値効果（A/B）の「下位/上位」ラジオは**維持**（所持個体がどちらの値かを選ぶ意味がある）。
  - 無欠の2効果同時付与はサブ入力2つ→固定値2つの表示のみに。系列は10%固定表示。
  - 数値なし効果（即時発動化）は「効果のみ」表示（現行どおり）。
- データ: `value_text` は保存構造として維持（Excel互換のため）。登録・更新時に必ずマスタ値から生成する。
- 既存データの扱い（**仮決め**）: 読込時にマスタ値と不一致でも上書きせず保持し、編集画面を開いて保存した時点でマスタ値へ正規化する。
- Excel読込（**仮決め**）: 数値列は参考情報とし、効果+等級から再計算した値と異なる場合は「警告」を出して取り込む（エラーにはしない）。
- docs改訂対象（本改修確定後）: `02_要件定義書.md` REQ-RULE-005、`03_画面設計書.md` §5(4番)、`04_データ設計書.md` §5 の「手編集可」記述。
- 完了条件: 数値欄が編集不可で、タイプ→効果→等級（→A/B）選択だけで値が確定する。

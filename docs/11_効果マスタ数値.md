## 1. 出典・確定状況

### 判明した仕様（まな確認済み）

- **「A / B」表記** … 同一等級内で **取りうる2つの値（A か B）**。例: 攻撃力増加 `5 / 10` は「5 または 10」（強化段階による下位/上位と推定）。`value_a`（下位）/ `value_b`（上位）で保持する。※「別々の2効果」ではなく、いずれか1つを取る値。
- **カンマ区切り（無欠のみ）** … `20 , 3%` のように **1効果へ2種類の効果が同時付与** される別パターン（`/`＝2値とは別物）。`flawless_debuff` のみ。
- **煌めく（燦爛）** … 効果は **混沌にのみ存在** する（深淵・太古は無し）。
- **秘伝タイプ名** … 設計書の名称（煌めく・整った・微か）を正式名として維持する。画像側の表記（燦爛・整頓・希微）は表示エイリアスとして併記のみ。
- **系列タイプ** … この表には無いが独立タイプとして存在。**深淵で10%統一**。太古・混沌の値は未確認（§4 要確認）。
- **`-`（ダッシュ）** … その等級には効果値が存在しない（登録・装着対象外）。
- **数値なしの効果**（例: 即時発動化）… 値を持たず効果の有無だけ。

## 4. 要確認・残課題

- ~~**系列タイプの効果名**~~ … 解決（2026-07-13）。アール／セルト／アヒブ／ラブリフの4種（§3参照）。
- **`radiant_instant`（即時発動化）** … 値を持たない効果。UIで「効果あり/なし」だけ表示する扱いで良いか。
- **「A か B」2値の位置づけ** … 同一等級内で取りうる2値と確認済み（例: `5 / 10`＝5または10）。下位/上位が何で決まるか（強化段階か個体差か）は推定。UIで範囲表示（5〜10）にするか、両値を出すかは実装前に決める。
- **等級ごとの装着可否** … 深淵にダッシュ（`-`）がある効果（守護のダメージ減少発動など）や、混沌のみの煌めくは、その等級の秘伝が存在しない前提。登録フォームで等級選択を制限するか要検討。

## 5. データ構造への反映方針

- `effect_master` に等級別の既定値を持たせる（深淵/太古/混沌 × value_a / value_b）。
- ユーザーの所持秘伝（`inventory_items`）は `effect_id` と `rarity` を選ぶと、対応する既定値が自動で入る。
- 既定値はあくまで初期値で、`value_text` による手編集は引き続き可能（ゲーム更新・個体差に対応）。
- 詳細なスキーマ追記は `04_データ設計書.md` §effect_master を参照。

# スキル秘伝HP テスト・受け入れ基準

## 1. 目的

実装後に「動いているように見える」だけで完成とせず、ゲームルール・保存・復元・画像・公開条件を確認する基準を先に定める。

## 2. マスタ検証

| ID | 確認内容 | 合格条件 |
| :---- | :---- | :---- |
| M-001 | クラス数 | 承認済みクラスが重複なく読み込まれる |
| M-002 | 現行スキル数 | 各クラスで上段4＋下段13を確認できる |
| M-003 | LS不足修正 | LS下段13に「神獣の加護」が存在する |
| M-004 | 装着可能数 | 上段4＋下段1〜12の計16スキルが対象 |
| M-005 | 対象外 | 下段13番は0枠で「秘伝対象外」と表示される |
| M-006 | 固定枠 | 装着対象スキルは4枠を持つ |
| M-007 | 枠タイプ | Excelの固定タイプと一致する |
| M-008 | 効果候補 | 7秘伝タイプの効果一覧を読み込める |
| M-009 | ID | クラス・スキル・効果IDが重複しない |

## 3. 所持秘伝

| ID | 操作 | 合格条件 |
| :---- | :---- | :---- |
| I-001 | 新規登録 | タイプ・効果・等級・所持数を保存できる |
| I-002 | 数値未入力 | 数値空欄でも登録できる |
| I-003 | 数値編集 | `%`、秒、複数値を文字列で保存できる |
| I-004 | 所持数 | 1以上だけを保存できる |
| I-005 | 同一候補 | 同条件の所持品へ統合候補を表示する |
| I-006 | 検索 | タイプ・効果・等級で絞り込める |

## 4. 装着

| ID | 操作 | 合格条件 |
| :---- | :---- | :---- |
| E-001 | 対応枠へ装着 | タイプ一致時だけ装着できる |
| E-002 | 非対応枠 | タイプ不一致では装着できない |
| E-003 | 未所持 | 所持登録のない秘伝を装着できない |
| E-004 | 所持数1 | 1個を同じ編成の複数スキルへ装着できない |
| E-005 | 所持数2 | 同じ秘伝を2スキルへ1個ずつ装着できる |
| E-006 | 解除 | 解除後に残数が戻る |
| E-007 | 移動 | 使用中の秘伝を確認つきで別スキルへ移動できる |
| E-008 | 数量減少 | 使用数を下回る数量へ無警告で変更できない |
| E-009 | 対象外スキル | 下段13へ装着操作を行えない |

## 5. 編成・保存

| ID | 操作 | 合格条件 |
| :---- | :---- | :---- |
| B-001 | 自動保存 | 再読込後も所持品・編成が残る |
| B-002 | 複数編成 | 同じクラスに複数編成を保存できる |
| B-003 | クラス切替 | 他クラスへ移動しても編成が失われない |
| B-004 | 保存失敗 | 容量・ブラウザ制限時に警告する |
| B-005 | データ削除 | 確認なしに全データを消せない |

## 6. Excel

| ID | 操作 | 合格条件 |
| :---- | :---- | :---- |
| X-001 | 書き出し | 必須4シート以上を含むExcelを生成できる |
| X-002 | 同一PC復元 | 書き出したExcelで同じ状態を復元できる |
| X-003 | 別PC復元 | 空の環境へ所持品と編成を復元できる |
| X-004 | 置換 | 現在データを退避後に置換できる |
| X-005 | 追加統合 | 競合内容を確認して取り込める |
| X-006 | 不正列 | 必須列不足を取込前に検出する |
| X-007 | 不正数量 | 0・負数・文字列数量をエラーにする |
| X-008 | 旧マスタ | 警告つきで保持または修正できる |

## 7. 画面・画像

| ID | 確認内容 | 合格条件 |
| :---- | :---- | :---- |
| U-001 | クラス選択 | アイコンと名称が対応する |
| U-002 | スキル画像 | 選択クラス分が正しく表示される |
| U-003 | 背景ランダム | 再起動で候補が変わる |
| U-004 | クラス背景 | 選択後は対象クラス画像だけを使う |
| U-005 | 背景失敗 | 画像なしでも操作できる |
| U-006 | 選択状態 | 色と文字で選択中を判別できる |
| U-007 | PC幅 | 1024px以上で主要操作が隠れない |
| U-008 | キーボード | 主要ボタンへTab移動できる |

## 8. 公開前チェック

- 非公式ファンサイト表記が常時確認できる。
- 公式リンクが正しい。
- ユーザーデータを外部送信していない。
- ゲームID・パスワード入力欄がない。
- 最新のファンコンテンツガイドを再確認した。
- 画像とクラス・スキル名の対応を確認した。
- マスタ・保存形式のバージョンを表示している。

## 9. 完成判定

MVP完成は、重大度「高」の問題が0件で、上記の必須テストがすべて合格し、Operatorが動作確認を完了した時点とする。合格前はOutputへ完成品として保存しない。
